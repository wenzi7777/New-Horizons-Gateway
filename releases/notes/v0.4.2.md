# New Horizons Gateway v0.4.2

## Highlights

- Closing the Gateway TUI now stops the Gateway runtime as well. The foreground
  launcher signals the backend loop to shut down and waits for the runtime
  thread to exit instead of leaving the service running behind the closed UI.
- Fixed the shutdown logging race that produced `RuntimeError: App is not
  running` when late `/api/status` request logs arrived after Textual had
  already stopped.
- Fixed the threaded TUI runtime startup path by moving signal registration
  behind a main-thread guard and running the Gateway runtime with an explicit
  stop event.
