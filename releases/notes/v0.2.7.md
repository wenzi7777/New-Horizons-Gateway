# New Horizons Gateway v0.2.7

## Fixes

- **Fixed gateway crash on "Serve This Device"**: `UDPCommandDispatcher._send_entry` was
  re-raising `send_datagram` exceptions, which propagated through `service()` into the main
  process loop and killed the gateway.  Failed sends are now absorbed — the error is recorded,
  the failed attempt is counted toward the delivery-timeout limit, and the process stays alive.

- **Fixed Transfer / Switch Gateway failing to complete**: When the upstream server marked a
  claim as failed before the device's first `findme_discover` arrived, `DiscoveryResponder`
  sent an offer without a `claim_id`.  The firmware's `claimId_` check then rejected every
  offer from the preferred gateway with `findme_transfer_offer_mismatch`, and the device fell
  back to its original gateway after the 30-second transfer TTL.  The responder now mirrors
  the device's own `claim_id` from the discover packet when we are the preferred gateway and
  our internal claim record is absent, so the firmware check passes.

## Verification

- Gateway unit and integration test suite (69 tests).
- New regression tests: `test_service_does_not_raise_when_send_datagram_fails`,
  `test_send_command_to_fails_gracefully_when_send_datagram_raises`,
  `test_offer_mirrors_discover_claim_id_when_claim_record_is_absent`,
  `test_offer_does_not_mirror_claim_id_when_not_preferred_gateway`.
