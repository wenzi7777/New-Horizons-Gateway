# New Horizons Gateway v0.1.6

## Changes

- Fixed UDP command dispatch: binary heartbeat and stream packets now register the UDP session using `CONTROL_PORT` (22345) instead of the stream source port (13250). Commands sent from the upstream server are now delivered to the device's control socket where they are actually processed.
- Removed the direct TCP control path (`send_control_command`) from the gateway command handler. The gateway now routes all commands exclusively via UDP, matching the NHO/Arduino/1 protocol design. The Desktop backend continues to handle its own TCP control path independently.
