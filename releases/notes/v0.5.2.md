# New Horizons Gateway v0.5.2

## Changes

### Gateway periodic self-check for updates

- The Gateway now independently checks GitHub for new versions at a
  configurable interval (default: 600 seconds / 10 minutes), without relying
  solely on the Desktop backend's WebSocket handshake notification.
- New version detection is added to `main.py`'s main loop, calling
  `update_manager.check(force=True)` periodically.
- `update_manager.check()` now syncs the manifest version into
  `latest_gateway_version` and `required_update` when a newer version is found,
  so the full update flow (WebUI overlay, `/api/status`) works correctly
  regardless of whether the Desktop backend sent a version signal.

This means: after pushing a new release to GitHub, existing Gateway instances
will autonomously discover the update within `auto_check_interval_sec` seconds
(10 min by default) and prompt the user — no reconnect or manual refresh needed.
