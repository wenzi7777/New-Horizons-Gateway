# New Horizons Gateway v0.3.0

## Highlights

- The Desktop backend WebSocket now reports `latest_gateway_version` in
  `gateway_hello_ack`, so Gateway clients can detect required updates
  immediately after connecting.
- The Gateway now enters a mandatory OTA mode when the server reports a newer
  Gateway version. Normal settings, discovery, allow/reject, and serve actions
  are blocked with `409 update_required` until the update is applied.
- The WebUI now includes a more prominent `Update Center`, changelog display,
  server-vs-manifest version breakdown, and a full-screen update overlay for
  forced upgrades.
- Windows startup is now standardized on `python scripts/start.py` and
  `python scripts/stop.py`. The old PowerShell launchers were removed, and the
  dedicated Gateway console now shows a clear banner instead of an empty shell.
