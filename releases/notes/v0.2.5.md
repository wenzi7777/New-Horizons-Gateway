# New Horizons Gateway v0.2.5

## Changes

### Package Installation via `pyproject.toml`

- Replaced the `PYTHONPATH` workaround in startup scripts with a proper
  `pip install -e .` using the new `pyproject.toml`. The `newhorizons_gateway`
  package is now correctly registered in the virtual environment on first run,
  eliminating import failures on fresh installations.

### Cross-Platform Python Launcher

- Added `start.py` / `stop.py` for macOS, Linux, and Windows systems with
  Python 3.9+ but without bash or PowerShell. The launcher creates a venv,
  installs dependencies, checks ports, spawns the gateway in the background,
  and writes a PID file — mirroring the behaviour of `start.sh` / `start.ps1`.
- Fixed a Windows PowerShell parse error in `start.ps1` caused by a UTF-8 em
  dash (U+2014) being decoded as a Windows-1252 right curly-quote. Replaced
  with an ASCII hyphen.
