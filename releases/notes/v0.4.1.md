# New Horizons Gateway v0.4.1

## Highlights

- Simplified Gateway startup to Python-only entrypoints. The packaged release
  now uses `python scripts/start.py` and `python scripts/stop.py` across macOS,
  Linux, and Windows, with the old shell wrappers removed.
- Fixed the macOS launch path so `python scripts/start.py` now enters the same
  foreground TUI flow as the Windows dedicated console, instead of silently
  backgrounding the Gateway.
- Hardened the TUI event log widget for packaged environments by switching the
  scrolling event panel to Textual `Log`, reducing startup-time compatibility
  failures.
- Fixed the startup crash where the TUI launched `newhorizons_gateway.main`
  from a worker thread and hit `ValueError: signal only works in main thread of
  the main interpreter`.
