# New Horizons Gateway v0.2.3

## Bug Fixes

- Fixed a bug where a device connected to one Gateway could not be transferred to another Gateway on the same LAN. Clicking "Transfer to this gateway" previously created a claim but never notified the device to switch — the device remained attached to the original Gateway until the claim timed out (30 s). The fix sends a `findme_switch_gateway` UDP command directly to the device's last known IP (port 13250) as soon as "Transfer" is clicked, and retries every 3 seconds while the claim is active. This triggers the device to restart FindMe discovery with the target Gateway's ID as `preferred_gateway_id`, allowing the new Gateway's claim-boosted offer (priority +10 000) to win and complete the handover.
