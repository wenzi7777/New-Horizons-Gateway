# New Horizons Gateway v0.2.4

## Fixes

- Run Gateway directly on the LAN host so FindMe and UDP command replies use the
  device's real IP address instead of Docker Desktop NAT addresses.
- Send `findme_switch_gateway` through the standard UDP command envelope with a
  stable claim request ID, ACK handling, retry, and claim expiry.
- Keep a claim pending until the target Gateway receives the device heartbeat;
  a switch command result alone no longer marks the device attached.
- Prevent delivery timeout from downgrading a claim that has already attached.
- Remove Docker Gateway assets and add startup port checks for UDP `22346`, UDP
  `13250`, and TCP `5052`.

## Verification

- Gateway unit and integration test suite.
- Dual-Gateway handover integration test covering old-Gateway rejection,
  matching target claim offer, and heartbeat attachment.
- Host-only release archive inspection.
