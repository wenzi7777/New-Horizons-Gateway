# New Horizons Gateway v0.1.9

## Changes

- Added reassembly of chunked device command results. Large command responses (e.g. `status`, ~2.5 KB) exceed a single UDP datagram and were lost to IP fragmentation. Firmware v0.6.5+ now splits oversized results into `result_chunk` frames keyed by `request_id`; the gateway collects the chunks and reassembles the full result before forwarding it upstream. Small results are unaffected and still travel inline. New module `result_chunks.py` (`ResultChunkReassembler`) with a 10s buffer TTL handles out-of-order chunks and purges incomplete buffers.
