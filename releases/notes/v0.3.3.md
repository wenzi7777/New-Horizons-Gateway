# New Horizons Gateway v0.3.3

## Highlights

- Replaced the ad-hoc Windows console redraw loop with a proper `Textual` TUI.
  Gateway now keeps the live status block fixed at the top while recent events
  scroll independently below it.
- Kept the live status header connected to the real runtime snapshot, so fields
  such as `Enabled`, upstream connectivity, required update state, and latest
  version continue to refresh while the process is running.
- Preserved the dedicated Windows Gateway console flow: the second window is
  now explicitly the `New Horizons Gateway` console instead of an apparently
  empty shell.
