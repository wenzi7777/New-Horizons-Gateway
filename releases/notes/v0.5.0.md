# New Horizons Gateway v0.5.0

## Highlights

- Rebuilt Gateway OTA around a minimal bootloader plus A/B runtime slots. OTA
  no longer stages into `pending-release` and no longer overwrites the live app
  tree while it is running.
- Added explicit local boot state and health contracts. New versions now switch
  through an inactive slot and must report both a healthy `health.json` payload
  and a ready local WebUI port before the bootloader commits the upgrade.
- Added automatic rollback when the pending slot fails first boot health
  confirmation, crashes, or never brings up the WebUI in time.
- Split startup into a stable bootloader entrypoint (`scripts/start.py`) and a
  slot runtime entrypoint (`scripts/start_runtime.py`) so the update substrate
  stays outside the OTA payload it manages.
- Updated the Update Center to report active slot, pending slot, and boot
  phase, matching the new switch-and-rollback flow.
