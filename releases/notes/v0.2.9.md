# New Horizons Gateway v0.2.9

## Fixes

- Removed the remaining Docker-specific Gateway distribution path. The Gateway
  is now documented and released as a host-native relay only.
- Enabled host-native Gateway self-update without the old
  `NEWHORIZONS_GATEWAY_ALLOW_SELF_UPDATE` gate. The WebUI update flow now
  matches the supported runtime model.
- Included `pyproject.toml` in the self-update allowlist so a restarted Gateway
  re-installs against the updated editable package metadata after applying an
  update.

## Verification

- Gateway unit and static release metadata test suite.
- Release zip audit confirms there are no Docker files in the distributable
  artifact.
