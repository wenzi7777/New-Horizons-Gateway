# New Horizons Gateway v0.2.6

## Features

- **Discover Devices across Gateways**: Added `findme_probe` broadcast mechanism so
  Gateway B can discover devices already attached to Gateway A on the same LAN.
  Clicking "Discover Devices" sends a probe; attached devices respond with their
  current gateway ID so the receiving gateway records the IP without stealing them.
- **Transfer / Serve This Device**: `findme_switch_gateway` is now reliably routed
  even when the target device has never sent a FindMe to this gateway.  The probe
  response populates the peer address so the switch command can be delivered.
- **Cross-platform Python start/stop scripts**: Added `scripts/start.py` and
  `scripts/stop.py` that work on macOS, Linux, and Windows without requiring bash
  or PowerShell, using only the Python standard library.
- **Self-update restart support**: All start scripts now export
  `NEWHORIZONS_GATEWAY_RESTART_COMMAND` so `update_manager.py` can restart the
  gateway after a successful self-update.

## Fixes

- Fixed `ModuleNotFoundError: No module named 'newhorizons_gateway'` on first run
  by adding `pyproject.toml` with an editable install and running
  `pip install -e .` in all start scripts.
- Fixed Windows PowerShell `start.ps1` parse error caused by a UTF-8 em dash
  (U+2014) being misread as Windows-1252; replaced with an ASCII hyphen.

## Verification

- Gateway unit and integration test suite.
- Cross-platform start/stop tested on macOS (bash and Python).
- Dual-Gateway discover and transfer integration test.
